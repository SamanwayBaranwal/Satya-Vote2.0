# 🪷 Satya Vote 2.0 — *Ek Vote, Ek Sach*

A transparent, tamper-proof, decentralized voting platform on Ethereum.

- **Frontend:** React + Vite + Tailwind CSS
- **Blockchain:** Solidity (`SatyaVote.sol`) on the Sepolia testnet
- **Wallet:** RainbowKit + Wagmi + viem/ethers v6
- **Dev:** Hardhat

## Smart contract features

- **Admin** = deployer. Can approve voters, create elections, add candidates, start/end.
- **Voter whitelist** — only `approveVoter`'d addresses can vote (batch approve supported).
- **One vote per wallet per election**, enforced on-chain.
- **Time-locked** elections (start/end timestamps) — votes rejected outside the window.
- **Immutable on-chain tallies**; winner computed on-chain.
- **Events** emitted for every action (`VoterApproved`, `ElectionCreated`, `CandidateAdded`, `VoteCast`, `ResultsDeclared`, …) for full audit.

## Getting started

### 1. Contracts

```bash
npm install
npm test                 # run the test suite
npx hardhat node         # (terminal 1) local chain
npm run deploy:local     # (terminal 2) deploy + seed demo election, export ABI to frontend
```

### 2. Frontend

```bash
cd frontend
npm install
cp .env.example .env     # add your WalletConnect projectId
npm run dev              # http://localhost:5173
```

### 3. Deploy to Sepolia

```bash
cp .env.example .env     # fill SEPOLIA_RPC_URL, PRIVATE_KEY, ETHERSCAN_API_KEY
npm run deploy:sepolia   # writes the sepolia address into frontend/src/contracts/addresses.json
npm run verify:sepolia <DEPLOYED_ADDRESS>
```

The deployer wallet automatically becomes admin and sees the **Admin** tab in the app.
